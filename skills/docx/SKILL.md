# Document Creator

Create polished, professional Word documents for business, technical, and administrative purposes.

## When to Use This Skill

Use this skill when you need to:
- Write event proposals or speaker applications
- Create partnership or sponsorship documents
- Draft contracts and agreements (as templates, not legal advice)
- Build technical documentation
- Generate invoices and statements
- Write formal reports or one-pagers
- Create onboarding materials

## Document Types & Templates

### Event Proposal
**Structure:**
1. Executive Summary (1 paragraph)
2. Event Overview (date, location, audience)
3. Value Proposition (why sponsor/partner)
4. Deliverables (what they get)
5. Investment Levels (if applicable)
6. Timeline
7. Contact Information

### Speaker Application
**Structure:**
1. Talk Title & Abstract (150 words max)
2. Speaker Bio (100 words)
3. Target Audience
4. Key Takeaways (3-5 bullets)
5. Technical Requirements
6. Previous Speaking Experience

### Technical Documentation
**Structure:**
1. Overview/Introduction
2. Prerequisites
3. Getting Started
4. Core Concepts
5. API Reference / Usage Examples
6. Troubleshooting
7. Contributing Guidelines

### Invoice Template
**Structure:**
1. Header (Your info, logo if available)
2. Bill To (Client info)
3. Invoice Details (number, date, due date)
4. Line Items (description, quantity, rate, amount)
5. Subtotal, Tax, Total
6. Payment Instructions
7. Terms & Conditions

### Partnership One-Pager
**Structure:**
1. Headline value proposition
2. Problem statement (2-3 sentences)
3. Solution overview
4. Key benefits (3-4 points)
5. Success metrics / social proof
6. Next steps / CTA

## Formatting Standards

### Typography
- **Headers:** Bold, 14-16pt
- **Body:** 11-12pt, 1.15-1.5 line spacing
- **Fonts:** Professional (Calibri, Arial, or similar)

### Layout
- 1-inch margins
- Left-aligned body text
- Consistent heading hierarchy
- Page numbers for multi-page docs

### Professional Elements
- Date on all documents
- Version number for technical docs
- Contact information
- Clear section breaks

## Subcontractor/Consultant Specifics

For consultant work, always include:
- Clear scope of work
- Deliverables with dates
- Payment terms (Net 15, Net 30)
- Revision policy
- Intellectual property terms

## File Output

Generate .docx files using python-docx with:
- Proper heading styles (Heading 1, 2, 3)
- Table formatting for structured data
- Page breaks between major sections
- Headers/footers where appropriate
